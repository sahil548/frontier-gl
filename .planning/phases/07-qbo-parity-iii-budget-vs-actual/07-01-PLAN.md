1. Add Budget model to prisma/schema.prisma:
   - Fields: id (cuid), entityId (String), accountId (String), year (Int), month (Int, 1-12 calendar month), amount (Decimal @default(0) @db.Decimal(19,4)), createdAt, updatedAt
   - Relations: entity -> Entity, account -> Account
   - Unique constraint: @@unique([entityId, accountId, year, month])
   - Index: @@index([entityId, year])
   - Map to "budgets" table
   - Add `budgets Budget[]` relation field to Entity model and Account model

2. Run `npx prisma db push` to sync schema.

3. Create src/lib/utils/fiscal-year.ts:
   - Export `getFiscalYearMonths(fiscalYearEnd: string, fiscalYear: number): { year: number; month: number; label: string }[]`
   - fiscalYearEnd is "MM-DD" format. For FYE "12-31", fiscal year 2025 = Jan 2025 - Dec 2025. For FYE "03-31", fiscal year 2025 = Apr 2024 - Mar 2025.
   - First month of fiscal year is (fyeMonth + 1). If fyeMonth is 12, first month is Jan of same year. If fyeMonth is 3, first month is Apr of prior year.
   - label: short month name (Jan, Feb, etc.)
   - Export `getFiscalYearDateRange(fiscalYearEnd: string, fiscalYear: number): { startDate: Date; endDate: Date }` for report queries.

4. Create src/validators/budget.ts:
   - `budgetLineSchema`: z.object({ accountId: z.string().cuid(), year: z.number().int().min(2000).max(2100), month: z.number().int().min(1).max(12), amount: z.string() (string to preserve decimal precision) })
   - `budgetUpsertSchema`: z.object({ fiscalYear: z.number().int(), budgets: z.array(budgetLineSchema) })
   - `budgetCsvRowSchema`: z.object({ accountNumber: z.string().min(1), period: z.string().regex(/^\d{1,2}\/\d{4}$/), amount: z.string().refine(s => !isNaN(parseFloat(s))) })

<task type="auto">
  <name>Task 2: Budget CRUD and CSV import API endpoints</name>
  <files>src/app/api/entities/[entityId]/budgets/route.ts, src/app/api/entities/[entityId]/budgets/import/route.ts</files>
  <action>
1. Create src/app/api/entities/[entityId]/budgets/route.ts:

   GET handler:
   - Auth: use Clerk auth() to get userId, call findAccessibleEntity(entityId, userId). Return 403 if null.
   - Query param: fiscalYear (required, number). Also read entity.fiscalYearEnd.
   - Use getFiscalYearMonths to determine which (year, month) pairs belong to this fiscal year.
   - Query: prisma.budget.findMany where entityId AND (year, month) IN the fiscal year months. Include account (select id, number, name, type).
   - Filter to only INCOME and EXPENSE accounts.
   - Return: { data: Budget[] } with serializeDecimal on amount fields.

   PUT handler:
   - Auth: same as GET. Also check role is Owner or Editor (not Viewer) via entityAccess.
   - Body: validate with budgetUpsertSchema.
   - Use prisma.$transaction with array of prisma.budget.upsert calls. Where clause uses the entityId_accountId_year_month unique. Create sets all fields. Update sets amount only.
   - Filter out rows where amount is "0" or empty string (skip zero budgets to keep DB clean).
   - Return: { data: { upserted: count } }.

2. Create src/app/api/entities/[entityId]/budgets/import/route.ts:

   POST handler:
   - Auth: same pattern, Owner or Editor role.
   - Parse multipart form data to get CSV file (follow pattern from accounts/import/route.ts).
   - Use PapaParse to parse CSV text. Expect headers: account_number (or accountNumber), period (MM/YYYY), amount.
   - Validate each row with budgetCsvRowSchema. Collect errors with row numbers.
   - For valid rows: look up account by number + entityId. Skip rows where account not found or not INCOME/EXPENSE.
   - Parse period "MM/YYYY" into { year, month }.
   - Upsert all valid rows in a transaction (same pattern as PUT).
   - Return: { data: { imported: N, skipped: N, errors: [...] } }.
