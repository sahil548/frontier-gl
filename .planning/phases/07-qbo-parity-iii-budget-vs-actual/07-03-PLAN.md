  <action>
1. Add to src/lib/queries/report-queries.ts:

   New interface:
   ```typescript
   export interface BudgetVsActualRow {
     accountId: string;
     accountNumber: string;
     accountName: string;
     accountType: string; // "INCOME" | "EXPENSE"
     actual: number;
     budget: number;
     varianceDollar: number;
     variancePercent: number | null; // null when budget is zero
   }

   export interface BudgetVsActualData {
     incomeRows: BudgetVsActualRow[];
     expenseRows: BudgetVsActualRow[];
     totalIncome: { actual: number; budget: number; varianceDollar: number; variancePercent: number | null };
     totalExpenses: { actual: number; budget: number; varianceDollar: number; variancePercent: number | null };
     netIncome: { actual: number; budget: number; varianceDollar: number; variancePercent: number | null };
   }
   ```

   New function `getBudgetVsActual(entityId: string, startDate: Date, endDate: Date)`:
   - Use raw SQL (Prisma.sql tagged template) following the existing getIncomeStatement pattern.
   - Query: SELECT from accounts LEFT JOIN journal_entry_lines (via journal_entries with status='POSTED' and date in range) LEFT JOIN budgets (aggregated by accountId where year*100+month falls in range).
   - Compute actuals: for INCOME accounts, credit - debit; for EXPENSE accounts, debit - credit (same convention as P&L).
   - Include a month's budget if any day of that month falls within the date range (full-month inclusion).
   - Filter to INCOME and EXPENSE account types only, active accounts only.
   - Order by account number.
   - In JS: compute varianceDollar and variancePercent per row.
     - Income: varianceDollar = actual - budget (positive = favorable)
     - Expense: varianceDollar = budget - actual (positive = favorable)
     - variancePercent = budget !== 0 ? (varianceDollar / Math.abs(budget)) * 100 : null
   - Group into incomeRows and expenseRows.
   - Compute section totals and net income (income actual - expense actual, income budget - expense budget, etc.).

   - Query params: startDate, endDate (ISO strings, required).
   - Parse dates, call getBudgetVsActual(entityId, startDate, endDate).
   - Return { data: BudgetVsActualData }.
