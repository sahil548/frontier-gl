**Accounting utilities** (`src/lib/utils/accounting.ts`):
- `AccountType` enum or union type: 'ASSET' | 'LIABILITY' | 'EQUITY' | 'INCOME' | 'EXPENSE'
- `getNormalBalanceSide(accountType: AccountType): 'DEBIT' | 'CREDIT'` -- Assets and Expenses are debit-normal, Liabilities/Equity/Income are credit-normal
- `formatCurrency(amount: number | string): string` -- formats to 2 decimal places with comma separators and dollar sign. Handle both number and string inputs (Prisma Decimal serializes to string).
- `isDebitNormal(accountType: AccountType): boolean` -- convenience wrapper

**Ledger queries** (`src/lib/queries/ledger-queries.ts`):
- Import prisma client from the project's established location (check existing codebase pattern -- likely `@/lib/prisma` or `@/lib/db`)
- Define `LedgerFilters` type: `{ startDate: Date; endDate: Date; memoSearch?: string; minAmount?: number; maxAmount?: number }`
- Define `LedgerRow` type: `{ date: Date; jeNumber: string; jeId: string; description: string; lineMemo: string | null; debit: number; credit: number; runningBalance: number }`
- `getBeginningBalance(entityId: string, accountId: string, beforeDate: Date): Promise<number>` -- sums all posted transactions for the account before the start date, using account type to determine sign convention
- `getLedgerTransactions(entityId: string, accountId: string, filters: LedgerFilters): Promise<{ beginningBalance: number; transactions: LedgerRow[] }>` -- uses PostgreSQL `$queryRaw` with `SUM() OVER (ORDER BY je.date, je.id ROWS UNBOUNDED PRECEDING)` window function for running balance. Filters by date range (inclusive both ends). If memoSearch provided, add `AND (je.description ILIKE ${'%' + memoSearch + '%'} OR jeli.line_memo ILIKE ${'%' + memoSearch + '%'})`. If minAmount/maxAmount provided, add `AND (jeli.debit BETWEEN min AND max OR jeli.credit BETWEEN min AND max)`. Only include transactions where `je.status = 'POSTED'`. Return both beginning balance and transaction rows. Convert Prisma Decimal results to numbers in the mapping step.
- `getAccountSummary(entityId: string, accountId: string): Promise<{ currentBalance: number; ytdDebits: number; ytdCredits: number }>` -- current balance from all posted transactions, YTD debits/credits from Jan 1 of current year

IMPORTANT: The running balance must respect the account's normal balance side. For debit-normal accounts (Asset, Expense), running balance = beginning + cumulative(debit - credit). For credit-normal accounts (Liability, Equity, Income), running balance = beginning + cumulative(credit - debit). Use `getNormalBalanceSide` from accounting utils.

**Trial balance queries** (`src/lib/queries/trial-balance-queries.ts`):
- Define `TrialBalanceRow` type: `{ accountId: string; accountNumber: string; accountName: string; accountType: AccountType; totalDebits: number; totalCredits: number; netBalance: number }`
- Define `ConsolidatedTrialBalanceRow` type extending TrialBalanceRow with `entityId: string; entityName: string`
- `getTrialBalance(entityId: string, asOfDate: Date): Promise<TrialBalanceRow[]>` -- aggregates all posted transactions through asOfDate grouped by account. Uses `SUM(CASE WHEN ...)` pattern. Only includes active accounts. Orders by account_number.
- `getConsolidatedTrialBalance(entityIds: string[], asOfDate: Date): Promise<ConsolidatedTrialBalanceRow[]>` -- same aggregation but across multiple entities, includes entity_id and entity name in results for tree-style grouping. Groups by account_number AND entity_id.
