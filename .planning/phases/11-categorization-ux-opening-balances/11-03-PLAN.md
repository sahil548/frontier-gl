<objective>
Opening balance JE auto-generation on holding creation and adjusting JE on balance edit

Purpose: When a user creates a holding with a non-zero balance, the system automatically generates and posts a journal entry (debit holding GL account, credit Opening Balance Equity for assets; reversed for liabilities). When the balance is later edited, an adjusting JE for the difference is created. This eliminates manual opening balance entry -- zero friction onboarding for new holdings.
Output: Opening balance lib module, subledger POST and PUT handlers extended, holdings form with date picker
</objective>

2. **Create opening balance module (src/lib/bank-transactions/opening-balance.ts):**
   - Export `determineJEDirection(accountType: string, holdingAccountId: string, obeAccountId: string): { debitAccountId: string, creditAccountId: string }`:
     - ASSET types: debit holdingAccountId, credit obeAccountId
     - LIABILITY types: debit obeAccountId, credit holdingAccountId
   - Export `computeAdjustment(oldBalance: number, newBalance: number): { amount: number, isIncrease: boolean } | null`:
     - Returns null if difference is 0 (skip JE)
     - Returns `{ amount: Math.abs(newBalance - oldBalance), isIncrease: newBalance > oldBalance }`
   - Export `async function findOrCreateOBEAccount(tx: PrismaTransactionClient, entityId: string): Promise<{ id: string }>`:
     - Query: `tx.account.findFirst({ where: { entityId, name: "Opening Balance Equity", type: "EQUITY" } })`
     - If found, return it
     - If not found: find next available number starting at 3900 (`tx.account.findFirst({ where: { entityId, number: "3900" } })`). If taken, try 3901, 3902, etc.
     - Create account with `{ entityId, number: nextAvailable, name: "Opening Balance Equity", type: "EQUITY", parentId: null }`
     - Create empty AccountBalance row: `tx.accountBalance.create({ data: { accountId: newAccount.id, balance: 0 } })`
     - Return created account
   - Export `async function generateOpeningBalanceJE(tx, params: { entityId, userId, holdingAccountId, holdingAccountType, balance, date }): Promise<string | null>`:
     - Guard: if balance === 0, return null
     - Call findOrCreateOBEAccount
     - Call determineJEDirection
     - Call generateNextEntryNumber(tx, entityId)
     - Create JE with status POSTED, postedBy userId, postedAt now
     - Create 2 line items (debit/credit per direction)
     - Update AccountBalance for both accounts using upsert with increment/decrement
     - Create audit trail entry
     - Return JE id
   - Export `async function generateAdjustingJE(tx, params: { entityId, userId, holdingAccountId, holdingAccountType, oldBalance, newBalance, date }): Promise<string | null>`:
     - Call computeAdjustment -- if null (no difference), return null
     - Call findOrCreateOBEAccount (should already exist from opening balance)
     - Determine direction: if isIncrease, same as opening balance direction. If decrease, reverse direction.
     - Create POSTED JE with description "Balance adjustment: {holdingName}"
     - Update AccountBalance for both accounts
     - Return JE id
  </action>
