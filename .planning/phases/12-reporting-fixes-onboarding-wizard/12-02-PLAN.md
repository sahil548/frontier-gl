2. **Refactor getCashFlowStatement** in `report-queries.ts`:
   - In the SQL query (around line 640), add `a."cashFlowCategory"::text AS cash_flow_category` to the SELECT clause
   - Replace the entire name-matching for-loop (lines 686-768) with field-based classification:
     ```
     for (const r of rows) {
       const mv = toNum(r.net_movement);
       if (mv === 0) continue;
       // Income/Expense: skip (captured in net income), except depreciation add-back
       if (r.account_type === "INCOME" || r.account_type === "EXPENSE") {
         if (r.account_type === "EXPENSE" && r.account_name.toLowerCase().includes("depreciation")) {
           operatingItems.push({ accountName: `Add back: ${r.account_name}`, amount: mv });
         }
         continue;
       }
       // Use cashFlowCategory field, default to OPERATING for null
       const category = r.cash_flow_category || "OPERATING";
       if (category === "EXCLUDED") continue;
       const item: CashFlowItem = { accountName: r.account_name, amount: -mv };
       switch (category) {
         case "INVESTING": investingItems.push(item); break;
         case "FINANCING": financingItems.push(item); break;
         case "OPERATING":
         default: operatingItems.push({ accountName: `Change in ${r.account_name}`, amount: -mv }); break;
       }
     }
     ```
   - Also update the cash balance query (around line 776) to use `a."cashFlowCategory" = 'EXCLUDED'` instead of `LOWER(a.name) LIKE '%cash%'` for identifying cash accounts
