<objective>
Close three wizard UX gaps from Phase 12 UAT:

1. **Return path from wizard (Test 2a — major):** After the wizard routes the user to `/accounts` (Customize step), there is no way back to the wizard. Add a persistent "Return to Setup" affordance in the authenticated header that appears whenever the current entity has an incomplete wizard AND the user is not already on `/onboarding/*`.
2. **Pre-existing entities show '0 of 4' banner (Test 12 — cosmetic):** Entities that predate the wizard feature (e.g., Three Pagodas, LLC with full COA + JEs + holdings) show `wizardProgress={}` which renders the banner as "0 of 4 steps complete". Fix by detecting substantive data (any non-template-placeholder accounts, any posted JEs, any holdings) on the GET handler and backfilling wizardProgress to all-complete the first time it is read.
3. **JE date mismatch (Test 11 — cosmetic):** Form shows `01/01/YYYY` (fiscal year start) but stored JE is `Dec 31 prior year`. UAT user confirmed this is confusing — fix by matching the stored date to the form date (post on fiscal year start, not prior-day). The existing wizard-side `generateOpeningBalanceJE` already serializes `jeDate.toISOString().split("T")[0]` — verify no Date-constructor timezone regression, and update the confirmation toast to echo the stored JE date back to the user.

Purpose: Close all three outstanding wizard/onboarding UX gaps so new users never get stranded, existing-entity users never see a noisy banner, and opening-balance JE posting matches the UX user sees.

- Server-side backfill in the wizard-progress GET endpoint for pre-existing entities

1. Create `src/lib/onboarding/wizard-progress.ts`:
   ```typescript
   export interface WizardProgress {
     coaComplete: boolean;
     holdingsComplete: boolean;
     balancesComplete: boolean;
     transactionsComplete: boolean;
     dismissedAt?: string;
   }

   export const WIZARD_DEFAULT: WizardProgress = {
     coaComplete: false,
     holdingsComplete: false,
     balancesComplete: false,
     transactionsComplete: false,
   };

   export function isWizardInProgress(progress: WizardProgress | null | undefined): boolean {
     if (!progress) return true;
     if (progress.dismissedAt) return false;
     const allComplete =
       progress.coaComplete && progress.holdingsComplete &&
       progress.balancesComplete && progress.transactionsComplete;
     return !allComplete;
   }

   export interface SubstantiveDataCounts {
     accountCount: number;
     postedJECount: number;
     holdingCount: number;
   }

   export function hasSubstantiveData(counts: SubstantiveDataCounts): boolean {
     return counts.accountCount > 5
       || counts.postedJECount > 0
       || counts.holdingCount > 0;
   }

   export function backfillCompleteProgress(): WizardProgress {
     return {
       coaComplete: true,
       holdingsComplete: true,
       balancesComplete: true,
       transactionsComplete: true,
     };
   }
   ```
