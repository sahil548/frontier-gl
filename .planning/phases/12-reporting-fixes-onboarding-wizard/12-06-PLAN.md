<objective>
Fix two CSV column-mapping UX gaps from Phase 12 UAT:

1. **Saved mapping reuse (Test 9 — major):** The server-side handler at `src/app/api/entities/[entityId]/csv-column-map/route.ts:99-104` only checks saved mappings when the client sends `sourceName`. The client (`column-mapping-ui.tsx:84-92`) never sends `sourceName` on subsequent imports, so the saved mapping lookup never fires and the badge always reads "Auto-detected". Fix: change the server lookup to match saved mappings by a normalized header fingerprint (entity + importType + headers set) when `sourceName` is absent, and pre-populate the source name from the matched record so the UI shows the original name.
2. **COA Import dialog layout (Test 7 — cosmetic):** When the Column Mapping panel opens inside `CoaImportDialog`, the dialog grows taller than the viewport and the DialogHeader description is clipped above the visible area. Fix: constrain the Dialog to `max-h-[90vh]` with vertical scroll, and make the DialogHeader sticky so the title remains visible while the mapping panel scrolls.

Purpose: Restore saved-mapping reuse (the main productivity win of Phase 12-04) and deliver a readable modal layout for repeated COA imports.
Output:
- Server-side header-fingerprint fallback in `csv-column-map` route
- `findMappingByHeaders` helper in the mapping store
- Layout fix on COA Import DialogContent with sticky header
- Tests covering fingerprint lookup (3+ new cases in `column-mappings.test.ts`)
</objective>
