  <action>
Create src/lib/queries/consolidated-report-queries.ts with three functions:

**Helper: computeEliminations(entityIds, entityBreakdownMap)**
- Fetch active EliminationRule records where both entityAId and entityBId are in the entityIds array
- For each rule, look up the balance of accountA in entityA's data and accountB in entityB's data
- Convert both balances to absolute values before comparing (handles debit-normal vs credit-normal accounts)
- Elimination amount = min(abs(balanceA), abs(balanceB)) -- always positive input, stored as negative in EliminationRow.amount
- If abs(balanceA) != abs(balanceB) (epsilon 0.005), flag as mismatch with difference = abs(abs(balanceA) - abs(balanceB))
- Determine accountType from the account's type field
- IMPORTANT: Iterate over rules (not entities) to avoid double-counting (Pitfall 2 from research)

**findAccountBalance helper:** Given report data (IncomeStatementData or BalanceSheetData) and an accountId, search through all row arrays to find the matching row's netBalance. Return 0 if not found.

**getConsolidatedIncomeStatement(entityIds: string[], startDate: Date, endDate: Date, basis: 'accrual' | 'cash')**
1. Fetch entities from prisma.entity.findMany where id in entityIds, select id, name, fiscalYearEnd
2. Call getIncomeStatement for each entity in parallel via Promise.all
3. Build entityBreakdowns array with entity metadata + IncomeStatementData
4. Call computeEliminations passing income-relevant data
5. Compute totals: sum entity totals, then subtract elimination amounts grouped by INCOME vs EXPENSE accountType
6. Return ConsolidatedIncomeStatement

**getConsolidatedBalanceSheet(entityIds: string[], asOfDate: Date, basis: 'accrual' | 'cash')**
1. Same pattern: fetch entities, call getBalanceSheet per entity in parallel
2. Build entityBreakdowns
3. computeEliminations for balance sheet accounts
4. Compute totals: sum entity totals minus eliminations grouped by ASSET, LIABILITY, EQUITY
5. Return ConsolidatedBalanceSheet

**getConsolidatedCashFlow(entityIds: string[], startDate: Date, endDate: Date)**
1. Fetch entities, call getCashFlowStatement per entity in parallel
2. Aggregate per-section: combine operating items from all entities, investing items, financing items
3. For eliminations on cash flow: apply to line items (simpler approach -- add elimination items to the relevant section rather than a separate eliminations section, since cash flow is already categorized)
4. Actually, per user decision: keep the dedicated Eliminations section pattern consistent across all three statements. So compute eliminations the same way, and include them as a separate array.
5. Return ConsolidatedCashFlow with aggregated sections + eliminations

IMPORTANT: Use calendar date range always -- do NOT transform dates based on entity fiscal year (CONS-05 / Pitfall 5). Fiscal year end is display-only metadata in EntityBreakdown.
