    1. Create `src/middleware.ts` using the Clerk v5 pattern:
       ```typescript
       import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'

       const isPublicRoute = createRouteMatcher(['/sign-in(.*)', '/sign-up(.*)'])

       export default clerkMiddleware(async (auth, req) => {
         if (!isPublicRoute(req)) {
           await auth.protect()
         }
       })

       export const config = {
         matcher: [
           '/((?!_next|[^?]*\\.(?:html?|css|js(?!on)|jpe?g|webp|png|gif|svg|ttf|woff2?|ico|csv|docx?|xlsx?|zip|webmanifest)).*)',
           '/(api|trpc)(.*)',
         ],
       }
       ```
       This protects ALL routes except sign-in and sign-up. API routes under /api/ are also protected.

    2. Create the Prisma schema at `prisma/schema.prisma`:
       - datasource: postgresql, url from DATABASE_URL env, directUrl from DIRECT_DATABASE_URL env
       - generator: prisma-client-js

       Models:
       - **User**: id (cuid), clerkId (String @unique), email (String @unique), name (String?), createdAt, updatedAt. Has relation to Entity[].
       - **Entity**: id (cuid), name (String), type (EntityType enum), typeOther (String? -- for OTHER type free text), fiscalYearEnd (String -- "MM-DD" format, e.g. "12-31"), coaTemplate (CoaTemplate enum, default BLANK), isActive (Boolean, default true), createdById (String FK to User), createdAt, updatedAt. Indexes on [createdById] and [isActive].

       Enums:
       - **EntityType**: LP, LLC, CORPORATION, S_CORP, TRUST, FOUNDATION, PARTNERSHIP, INDIVIDUAL, OTHER
       - **CoaTemplate**: TEMPLATE, BLANK

       IMPORTANT for DI-01: This schema does not yet have money fields (those come in Phase 2 with Account and JournalEntry models). However, the schema structure establishes the pattern -- all future money fields MUST use `Decimal @db.Decimal(19,4)`. Add a comment in the schema noting this convention.

       IMPORTANT for DI-02: Entity model establishes the entity_id FK scoping pattern. All future financial tables MUST include `entityId String` with `@relation(fields: [entityId], references: [id])` to Entity. Add a comment noting this convention.

    3. Create Prisma singleton client at `src/lib/db/prisma.ts`:
       ```typescript
       import { PrismaClient } from "@prisma/client";

       const globalForPrisma = globalThis as unknown as {

       };

       export const prisma = globalForPrisma.prisma ?? new PrismaClient({
         log: process.env.NODE_ENV === "development" ? ["query"] : [],

       if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = prisma;
       ```
