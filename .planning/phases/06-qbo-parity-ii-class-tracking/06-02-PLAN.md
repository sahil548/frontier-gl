```prisma
model Dimension {
  id        String         @id @default(cuid())
  entityId  String
  name      String
  sortOrder Int            @default(0)
  isActive  Boolean        @default(true)
  tags      DimensionTag[]
  @@unique([entityId, name])
}

model DimensionTag {
  id          String    @id @default(cuid())
  dimensionId String
  code        String
  name        String
  isActive    Boolean   @default(true)
  dimension   Dimension @relation(...)
  lineAssignments JournalEntryLineDimensionTag[]
  @@unique([dimensionId, code])
}

model JournalEntryLineDimensionTag {
  id                 String           @id @default(cuid())
  journalEntryLineId String
  dimensionTagId     String
  @@unique([journalEntryLineId, dimensionTagId])
}
```

1. **JE Create endpoint** (POST route.ts):
   - After creating JournalEntryLine records, for each line that has dimensionTags entries, create JournalEntryLineDimensionTag junction records.
   - Validate one-tag-per-dimension constraint at the API level: for each line's dimensionTags, look up which dimension each tag belongs to (tag -> dimension mapping). If any dimension appears more than once per line, return 400 with error message.
   - Use nested create within the Prisma transaction: `lineItems: { create: lines.map(l => ({ ...lineData, dimensionTags: { create: Object.values(l.dimensionTags).filter(Boolean).map(tagId => ({ dimensionTagId: tagId })) } })) }`.

2. **JE Read endpoint** (GET [journalEntryId]/route.ts):
   - Include dimensionTags in the query: `lineItems: { include: { dimensionTags: { include: { dimensionTag: { include: { dimension: true } } } } } }`.
   - Serialize dimension tags in the response as `dimensionTags: { [dimensionId]: tagId }` per line for easy form consumption.

3. **JE Update endpoint** (PUT [journalEntryId]/route.ts):
   - On update (draft only), delete all existing JournalEntryLineDimensionTag records for the entry's lines, then recreate from the submitted dimensionTags. This is simpler than diffing and handles all add/remove/change cases.
   - Same one-tag-per-dimension validation as create.

4. **Existing entries without dimension tags** (CLASS-05): The dimensionTags field defaults to `{}` in the validator, so existing entries that don't submit dimension tags will have no junction records created -- they continue to work normally.
