   - `DimensionedIncomeStatementData`: `{ income: DimensionedReportRow[]; expenses: DimensionedReportRow[]; netIncome: number; tags: Array<{ id: string; name: string }> }`.

2. **Modify getIncomeStatement** to accept optional `dimensionId?: string` parameter:
   - When `dimensionId` is NOT provided: existing query unchanged, return IncomeStatementData as before (CLASS-05).
   - When `dimensionId` IS provided: modify the SQL to LEFT JOIN journal_entry_line_dimension_tags and dimension_tags, GROUP BY account + tag. The SQL should:
     - LEFT JOIN `journal_entry_line_dimension_tags jeldt ON jeldt."journalEntryLineId" = jel.id`
     - LEFT JOIN `dimension_tags dt ON dt.id = jeldt."dimensionTagId" AND dt."dimensionId" = $dimensionId`
     - Add `dt.id AS tag_id, dt.name AS tag_name` to SELECT
     - Add `dt.id, dt.name` to GROUP BY
   - Post-process: group raw rows by account, building tagBreakdown array. Rows where tag_id IS NULL go into the "Unclassified" bucket.
   - Also fetch the list of active tags for the dimension to provide column headers (include tags that have no data so columns are complete).
   - Return DimensionedIncomeStatementData.

3. **API route** (income-statement/route.ts):
   - Accept optional `dimensionId` query parameter.
   - Pass to getIncomeStatement. Return appropriate data shape.

   - When a dimension is selected: render column-per-tag layout. Table columns: Account | [Tag 1] | [Tag 2] | ... | Unclassified | Total. Each cell shows the net balance for that account+tag combo. Income section, then Expenses section, then Net Income row summing each column.
   - Cash/accrual toggle continues to work independently -- it's passed alongside dimensionId.
   - Horizontal scroll when many tag columns exist.
   - CSV export: when dimension view is active, export includes all tag columns.
