  <action>
1. **Prisma schema** -- Add three new models to schema.prisma:
   - `Dimension` with fields: id (cuid), entityId, name, sortOrder (default 0), isActive (default true), createdAt, updatedAt. Relations: entity (Entity), tags (DimensionTag[]). Unique constraint on [entityId, name]. Index on [entityId, isActive]. Map to "dimensions".
   - `DimensionTag` with fields: id (cuid), dimensionId, code (short alphanumeric), name, description (optional), isActive (default true), sortOrder (default 0), createdAt, updatedAt. Relations: dimension (Dimension, onDelete Cascade), lineAssignments (JournalEntryLineDimensionTag[]). Unique constraints on [dimensionId, code] and [dimensionId, name]. Index on [dimensionId, isActive]. Map to "dimension_tags".
   - `JournalEntryLineDimensionTag` with fields: id (cuid), journalEntryLineId, dimensionTagId. Relations: journalEntryLine (JournalEntryLine, onDelete Cascade), dimensionTag (DimensionTag). Unique on [journalEntryLineId, dimensionTagId]. Indexes on journalEntryLineId and dimensionTagId. Map to "journal_entry_line_dimension_tags".
   - Add `dimensions Dimension[]` to Entity model.
   - Add `dimensionTags JournalEntryLineDimensionTag[]` to JournalEntryLine model.
   - Run `npx prisma db push` to apply.

2. **Zod validators** (src/lib/validators/dimension.ts):
   - `createDimensionSchema`: name (string, 1-100 chars, required).
   - `updateDimensionSchema`: name (optional), isActive (optional boolean), sortOrder (optional number).
   - `createTagSchema`: code (string, 1-10 chars, alphanumeric, required), name (string, 1-100 chars, required), description (string, optional).
   - `updateTagSchema`: code (optional), name (optional), description (optional), isActive (optional boolean), sortOrder (optional number).
   - Export inferred types for all schemas.

3. **Unit tests** (src/lib/validators/dimension.test.ts):
   - Test createDimensionSchema accepts valid name, rejects empty.
   - Test createTagSchema accepts valid code+name, rejects empty code, rejects code >10 chars.
   - Test updateTagSchema accepts partial updates.

4. **API routes** -- Follow existing pattern (safeParse + successResponse/errorResponse):
   - `GET /api/entities/[entityId]/dimensions` -- List all dimensions with tags (include tags), ordered by sortOrder. Filter to isActive=true by default, accept `?all=true` to include inactive.
   - `POST /api/entities/[entityId]/dimensions` -- Create dimension. Validate with createDimensionSchema.
   - `GET /api/entities/[entityId]/dimensions/[dimensionId]` -- Single dimension with tags.
   - `PUT /api/entities/[entityId]/dimensions/[dimensionId]` -- Update dimension (name, isActive, sortOrder).
   - `DELETE /api/entities/[entityId]/dimensions/[dimensionId]` -- Soft delete (set isActive=false). Also deactivate all child tags.
   - `GET /api/entities/[entityId]/dimensions/[dimensionId]/tags` -- List tags for dimension.
   - `POST /api/entities/[entityId]/dimensions/[dimensionId]/tags` -- Create tag. Validate with createTagSchema.
   - `PUT /api/entities/[entityId]/dimensions/[dimensionId]/tags/[tagId]` -- Update tag.
   - `DELETE /api/entities/[entityId]/dimensions/[dimensionId]/tags/[tagId]` -- Soft delete (set isActive=false).
   - All routes scope by entityId from URL params. Use getOrCreateUser for auth.
  </action>
