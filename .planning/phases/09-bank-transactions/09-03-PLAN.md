  <action>
  First, install Plaid packages: `npm install plaid react-plaid-link`

  **src/lib/plaid/client.ts:**
  Create PlaidApi singleton using Configuration with basePath from PlaidEnvironments[process.env.PLAID_ENV || 'production']. Set PLAID-CLIENT-ID and PLAID-SECRET headers. Export `plaidClient`. Follow the research Pattern 1 code example exactly.

  **src/lib/plaid/encrypt.ts:**
  AES-256-GCM encryption/decryption using Node.js `crypto` module.
  - `encryptToken(plaintext: string): string` -- uses PLAID_ENCRYPTION_KEY env var (32-byte hex), generates random 16-byte IV, creates cipher with aes-256-gcm, returns `${iv_hex}:${authTag_hex}:${encrypted_hex}` concatenated string
  - `decryptToken(encrypted: string): string` -- splits on ':', creates decipher, sets auth tag, returns plaintext
  Throw clear errors if PLAID_ENCRYPTION_KEY is missing or invalid length.

  **src/lib/plaid/sync.ts:**
  Export `syncTransactions(connection: PlaidConnection): Promise<SyncResult>`.
  - Decrypt accessToken via decryptToken
  - Cursor-based loop: call plaidClient.transactionsSync with cursor, collect ALL pages into added/modified/removed arrays before persisting (CRITICAL: Plaid requirement)
  - On TRANSACTIONS_SYNC_MUTATION_DURING_PAGINATION error, restart loop from original cursor
  - Persist in a single Prisma transaction:
    - Added: create BankTransaction records with source PLAID, externalId = transaction_id. Invert Plaid amount sign (Plaid positive = outflow, we want negative for outflow). Skip if externalId already exists (duplicate)
    - Modified: update matching BankTransaction records by externalId
    - Removed: delete matching BankTransaction records by externalId (only if status is PENDING)
    - Update connection.syncCursor and connection.lastSyncAt
  - Auto-apply categorization rules to newly added transactions (fetch entity rules, call matchRule from Plan 01)
  - On ITEM_LOGIN_REQUIRED error: set connection.status = 'ERROR', connection.error = 'Bank requires re-authentication'
  - Return { added: number, modified: number, removed: number }

  **POST /api/plaid/create-link-token/route.ts:**
  - Authenticate via Clerk (getAuth)
  - Body: { subledgerItemId: string }
  - Check if an existing PlaidConnection with ERROR status exists for this subledgerItem -- if so, use update mode by passing access_token to linkTokenCreate
  - Call plaidClient.linkTokenCreate with: user.client_user_id = clerkUserId, client_name = 'Frontier GL', products = [Products.Transactions], country_codes = [CountryCode.Us], language = 'en'
  - Return { linkToken: response.data.link_token }

  **POST /api/plaid/exchange-token/route.ts:**
  - Body: { publicToken: string, subledgerItemId: string, institutionId?: string, institutionName?: string, accountId?: string }
  - Call plaidClient.itemPublicTokenExchange({ public_token: publicToken })
  - Encrypt access_token via encryptToken
  - Upsert PlaidConnection: create or update (for reconnect flow) with encrypted accessToken, itemId, institutionId, institutionName, plaidAccountId, status ACTIVE, error null
  - Trigger initial sync: call syncTransactions immediately
  - Return { success: true, transactionsSynced: result.added }

  **POST /api/plaid/sync/route.ts (manual Sync Now):**
  - Body: { subledgerItemId: string }
  - Find PlaidConnection by subledgerItemId, verify exists and is ACTIVE
  - Call syncTransactions(connection)
  - Return sync results

  **GET /api/cron/plaid-sync/route.ts:**
  - Verify CRON_SECRET: `request.headers.get('authorization') === 'Bearer ${process.env.CRON_SECRET}'`. Return 401 if invalid
  - Fetch all PlaidConnections with status ACTIVE
  - Process each sequentially (not parallel -- respects Plaid rate limits)
  - For each: try syncTransactions, catch errors and update connection status to ERROR
  - Return { synced: number, errors: number, results: [...] }

  - Test ITEM_LOGIN_REQUIRED sets connection status to ERROR
  - Test duplicate externalId is skipped (not double-inserted)
