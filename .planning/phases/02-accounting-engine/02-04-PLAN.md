From src/lib/validators/journal-entry.ts (Plan 01):
```typescript
export const journalEntrySchema = z.object({
  date: z.string().min(1),
  description: z.string().min(1).max(500),
  lineItems: z.array(lineItemSchema).min(2),
}).refine(/* total debits = total credits */);

export type JournalEntryFormValues = z.infer<typeof journalEntrySchema>;
```

API Endpoints (Plan 03):
- GET /api/entities/:entityId/journal-entries?status=DRAFT&page=1 -> { entries, total, page, limit }
- POST /api/entities/:entityId/journal-entries -> JE with lineItems (201)
- GET /api/entities/:entityId/journal-entries/:id -> JE with lineItems + audit
- PUT /api/entities/:entityId/journal-entries/:id -> updated JE (drafts only)
- DELETE /api/entities/:entityId/journal-entries/:id -> (drafts only)
- POST .../approve -> approved JE
- POST .../post -> posted JE (updates account balances)
- POST .../reverse -> new reversal draft JE (201)
- POST /api/entities/:entityId/journal-entries/bulk-post -> { posted, count }
- POST /api/entities/:entityId/journal-entries/bulk-approve -> { approved, count }

Account list for combobox (Plan 02):
- GET /api/entities/:entityId/accounts -> Account[] with balance
